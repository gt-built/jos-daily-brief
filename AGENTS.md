# AGENTS.md

## Project context
This project must be changed carefully and incrementally.

## Workflow rules
- Do not scan the full repository unless explicitly asked.
- Prefer reading only the files relevant to the task.
- Before editing, explain the intended change briefly.
- Make the smallest possible change.
- Do not rewrite complete files unless necessary.
- Prefer patches over full file output.
- After changes, summarize:
  1. files changed
  2. what changed
  3. how to test

## Coding rules
- Follow existing project structure.
- Follow existing naming conventions.
- Keep code simple and maintainable.
- Do not introduce new dependencies without asking.
- Add or update tests when logic changes.

## Usage control
- Keep responses concise.
- Do not repeat the full context.
- Do not paste large files back unless requested.
